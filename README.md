# LearnShelf
Here's a more detailed description for each of the 8 files:

 1. index.html
This is the main structural file that the browser actually opens. It contains the HTML skeleton for the entire site: the sticky header/navbar, the mobile slide-in menu, an empty `<main id="app">` container where all page content gets injected dynamically by JavaScript, plus the footer, the login/signup modal, and the online reader popup. It also links to all the CSS and JS files at the bottom, in a specific order. Think of this as the "shell" — it doesn't contain much visible content itself, since most of that is generated by app.js.

 2. css/style.css
This is the primary styling file containing the entire visual design system — color variables (including dark mode colors), typography, spacing, and the look of every component: buttons, cards, the navbar, hero section, resource cards, filters, modals, the reader, forms, footer, and animations like fade-ins and hover effects. This is the biggest CSS file since it covers desktop-first styling for the whole site.

 3. css/responsive.css
A smaller, focused stylesheet that only contains `@media` queries. It overrides styles from style.css at different screen widths so the layout adapts properly on tablets and phones — turning the navbar into a hamburger menu, stacking grid columns, converting the filter sidebar into a slide-in drawer, and resizing text/buttons for touch screens.

 4. js/data.js
A pure data file with no logic — just arrays and a function that builds them. It defines the list of subjects (Math, Physics, etc.), categories (Books, Notes, PDFs, etc.), and authors, then programmatically generates about 50 realistic resource objects (titles, ratings, views, downloads, pages, etc.) by combining these lists. This acts as the "database" for the whole site.

 5. js/auth.js
Handles everything related to user accounts, simulated entirely with localStorage (no real backend). It manages opening/closing the login and sign-up modal, validating email format and password length, checking password confirmation, registering new users, logging existing users in, and logging out. It also updates the header UI to show either "Login/Sign Up" buttons or a user avatar depending on login state.

 6. js/library.js
Manages a user's personal collections: favorites (bookmarked resources), My Library (saved resources), recently viewed items, and download history. It provides functions to add/remove items from these lists, updates the little heart badge count in the header, and saves everything to localStorage so it persists after refreshing the page.

 7. js/search.js
Powers two search-related features: the live search-as-you-type suggestions dropdown on the homepage hero search bar, and the full filtering/sorting engine on the Browse Library page (filtering by subject, category, author, type, rating; sorting by popularity, date, rating, or alphabetically; and switching between grid/list view).

 8. js/app.js
The largest and most central file — it's the "engine" that ties everything together. It contains the hash-based router (deciding which page to show based on the URL, like `#/home` or `#/resource/12`), functions that generate the actual HTML for every page (Home, Browse, Subjects, Resource Details, Favorites, Dashboard, etc.), the toast notification system, dark/light mode toggling, mobile menu behavior, scroll animations, animated counters, the back-to-top button, and the online reader popup logic (page turning, zoom, fullscreen). It also has the `DOMContentLoaded` listener that starts up the whole app when the page first loads.
